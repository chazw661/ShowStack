## Design Principles

Inherited from Phase 1, with one Phase 2 addition:

1. **Status at a glance.** Status indicators must be readable at 2 meters from a laptop screen without zooming.
2. **No false urgency.** Visual hierarchy must prevent amber from reading as critical and red from blending into the background.
3. **Minimal cognitive load.** An engineer in show mode must not hunt for information.
4. **Mode clarity.** When the engineer has suppressed alerts (Setup/Wrap mode), the UI must make that suppression impossible to miss. Silent failure is worse than a visible indicator.

## Interaction Contract

### Show Mode Toggle

- Click any segment: immediately updates mode state. Saves to `localStorage` key `nhm_show_mode`.
- Mode change POSTs to a new endpoint `POST /audiopatch/network-monitor/api/show-mode/` with `{"mode": "setup"|"show"|"wrap"}` to persist on the server for the active `MonitorSession`.
- Banner (`nhm-mode-banner`) shows/hides instantly on mode change via JS class toggle (no transition).
- Non-critical alert suppression is enforced server-side (the agent does not fire non-critical alerts when mode is setup/wrap). The frontend reflects server state.

### Settings Panel

- Clicking gear icon: adds `nhm-settings-panel--open` to panel, adds backdrop to DOM, traps focus inside panel.
- Clicking backdrop or pressing Escape: removes `nhm-settings-panel--open`, removes backdrop.
- Saving settings: `POST /audiopatch/network-monitor/api/snmp-settings/` with `{"community_string": "..."}`. On success: close panel, show inline success indicator (brief green border flash on gear button, 0.5s). On error: inline error message below save button (14px/400, `--accent-red`) â€” copy: "Couldn't save settings. Check your connection and try again."
- Adding manual switch: `POST /audiopatch/network-monitor/api/add-switch/` with `{"ip": "...", "label": "..."}`. On success: IP appears in configured switches list within panel. Input fields clear. On validation error (invalid IP): inline error below IP input (14px/400, `--accent-red`) â€” copy: "Enter a valid IP address." On duplicate IP error: inline error below IP input â€” copy: "{IP} is already being monitored."
- Focus trap: Tab key cycles focus within panel while open. Escape always closes.
- `aria-modal="true"` on panel, `role="dialog"`, `aria-label="Monitor Settings"`.

### Switch Card Expand/Collapse

- Click anywhere on card header row (not the port table rows): toggles expanded state.
- Expanded state: `nhm-card--expanded` class on `nhm-card`, `max-height` transition from 0 to 1200px.
- Expand arrow rotates: `nhm-domain-toggle` CSS `transform: rotate(180deg)` when expanded.
- `localStorage` key: `nhm_card_{device_id}_expanded` (same pattern as Phase 1).
- `aria-expanded` on the card header div toggles true/false.

### Port Table Data Refresh

- Port table data updates on the same AJAX poll cycle as device status (every 3 seconds).
- If port data arrives for an expanded card, the table rows update in-place via JS DOM manipulation. The table does not re-render entirely (avoids scroll position jump).
- Bandwidth percentage text color updates via class swap (`nhm-bw--ok`, `nhm-bw--warn`, `nhm-bw--crit`).

### SNMP State Transitions

| State | Card Collapsed | Card Expanded | Rollup Pill |
|-------|---------------|---------------|-------------|
| No SNMP config | status dot from ICMP only | "SNMP not configured" prompt | counts ICMP state |
| SNMP configured, responding | online dot, port summary | port table | counts up ports |
| SNMP unreachable | amber dot, "SNMP unreachable" | prompt + "Open Settings" | amber pill if any |

## Accessibility Minimums

Inherited from Phase 1, with Phase 2 additions:

- Show mode toggle buttons: `aria-pressed="true"` on active segment.
- Settings panel: `role="dialog"`, `aria-modal="true"`, `aria-label="Monitor Settings"`, focus trapped.
- Port table: `<thead>` with `<th scope="col">` for all column headers. Screen reader reads column header before value.
- Port status dots in table: `aria-label="{port_number}: up"` or `"down"` or `"unknown"`.
- Bandwidth cells: plain text percentage plus screen-reader-only text `<span class="sr-only">({threshold status})</span>` â€” e.g., "45% (ok)", "85% (warning)", "95% (critical)".
- Mode banner: `role="status"` so screen readers announce mode changes.
- All new buttons: visible focus ring via `outline: 2px solid var(--accent-blue); outline-offset: 2px`.
- Gear icon button: `title="Settings"` provides native tooltip on hover for sighted users; `aria-label="Open monitor settings"` provides full context for screen readers.

`.sr-only` utility class (add once to stylesheet):
```css
.sr-only {
    position: absolute;
    width: 1px;
    height: 1px;
    padding: 0;
    margin: -1px;
    overflow: hidden;
    clip: rect(0,0,0,0);
    white-space: nowrap;
    border: 0;
}
```
