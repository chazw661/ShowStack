# Phase 02 Plan 01: Schema Foundation (YAMAHA_COLOR_CHOICES + ConsoleImport) Summary

**One-liner:** Additive schema foundation adding Yamaha color palette storage to 4 channel models and a new ConsoleImport audit-snapshot model via migration 0153.

## What Was Built

This plan laid the schema foundation for Phase 2 Console CSV Import:

1. **`YAMAHA_COLOR_CHOICES`** — 10-entry constant defined above the `Console` class in `planner/models.py`, matching the key order of `YAMAHA_TO_HEX` in `planner/utils/reaper_export.py` exactly (`Off, Red, Orange, Yellow, Green, Sky Blue, Blue, Purple, Pink, White`).

2. **`color` field on 4 channel models** — `CharField(max_length=20, choices=YAMAHA_COLOR_CHOICES, default='Blue', blank=True)` added to:
   - `ConsoleInput` (line 845) — after `source_hardware` field
   - `ConsoleAuxOutput` (line 883) — after `omni_out` field
   - `ConsoleMatrixOutput` (line 902) — after `omni_out` field
   - `ConsoleStereoOutput` (line 918) — after `omni_out` field

3. **`ConsoleImport` model** — Immutable audit-history snapshot of one CSV/zip upload applied to a Console. 8 fields: `console` FK (CASCADE), `uploaded_by` FK (SET_NULL — preserves audit on user delete), `uploaded_at` (auto_now_add), `original_filename` (CharField), `raw_file` (FileField with `_import_upload_to` callable), `parsed_sections` (JSONField), `summary` (JSONField), `committed` (BooleanField). Meta: ordering `['-uploaded_at']`, verbose_name `"Console Import"`.

4. **`_import_upload_to` callable** — Returns `console_imports/{project_id}/{console_id}/{timestamp}-{safe_filename}` using `os.path.basename` to strip path components (path-traversal mitigation per RESEARCH Security Domain).

5. **Migration `0153_console_color_and_consoleimport.py`** — Auto-generated by `makemigrations`. Contains exactly 5 operations: 4 `AddField` (one per channel model) + 1 `CreateModel` (ConsoleImport). Zero `AlterField`/`RemoveField`/`RenameField`. Depends on `('planner', '0152_multitrack_session_track')`.

6. **`ConsoleImportAdmin`** — Subclasses `BaseEquipmentAdmin`. `list_display`, `list_filter`, `search_fields` configured. All meaningful fields in `readonly_fields` (immutable audit history). All three permission methods (`has_add`, `has_change`, `has_delete`) block Viewer-group users and permit superusers, mirroring `MultitrackSessionAdmin`. Registered on `showstack_admin_site` (never `admin.site`).

7. **`admin_ordering.py`** — Added `'consoleimport': 51` to `order_map`, immediately after `'multitracksession': 50`. Positions "Console Imports" at the bottom of the admin sidebar, adjacent to "Multitrack Sessions".
