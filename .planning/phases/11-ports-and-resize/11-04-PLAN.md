<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Engineer-typed port label → JointJS SVG `<text>` rendering | JointJS escapes text content (textContent into the `<text>` element); SVG `<text>` cannot execute script. |
| Cross-origin webfont → PNG export canvas | If a non-system font ever sneaks in, PNG export taints the canvas (security boundary at the export step). Mitigated by font-family lock in JS markup AND CSS Section 14. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-04-01 | Tampering (XSS via SVG text content) | portLabelMarkupForEdge generates `<text>` with engineer-controlled label | mitigate | JointJS renders engineer text via textContent into the SVG `<text>` element (never innerHTML). SVG `<text>` does not execute scripts. RESEARCH §Security verified this contract. |
| T-11-04-02 | Tampering (font taint of PNG export) | An admin theme or browser default could substitute a webfont for the system font, tainting PNG export | mitigate | Two-layer defense: JS markup sets `font-family` inline (`portLabelMarkupForEdge`), AND CSS Section 14 sets `.joint-port-label text { font-family: ... !important; }`. Both reference the same system-fonts-only stack. EXP-01 (Phase 10) regression covered. |
| T-11-04-03 | Information Disclosure | n/a | accept | No new endpoints; no new data flow. Pure presentation layer. |
