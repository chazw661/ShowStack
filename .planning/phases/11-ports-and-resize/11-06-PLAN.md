RESEARCH §Q7 Example 4 verbatim implementation (this plan inserts):

```javascript
function maybeAutoExpand(cell) {
  var current = cell.size();
  var min = computeMinSize(cell);
  var grew = false;
  var newW = current.width, newH = current.height;
  if (min.width  > current.width)  { newW = min.width;  grew = true; }
  if (min.height > current.height) { newH = min.height; grew = true; }
  if (!grew) return;
  if (window.__sfd.viewport.snapEnabled) {
    newW = Math.ceil(newW / 20) * 20;
    newH = Math.ceil(newH / 20) * 20;
  }
  cell.resize(newW, newH);
  showToast('Shape resized to fit ports.', 'info');
}
```

RESEARCH §Q7 key constraints:
- Use `Math.ceil` (NOT `Math.round`) for the snap step so snap rounding never shrinks below min.
- Do NOT call `cell.position()` — grow anchors top-left automatically.
- Toast suppressed if `!grew` — silent no-op when add/rename doesn't push above current size.
- `cell.resize()` fires `change:size` → Plan 11-05's listeners handle redistribute + autosave.

Pitfall 8 already mitigated by Plan 11-02: `addAuthoredPort` wraps the entire body (including maybeAutoExpand call) in `beginBatch/endBatch`, so a single Ctrl+Z reverses the port-add + auto-grow as one gesture.
