<objective>
Close GAP-11.4 — the only CSS-only gap from the 11-HUMAN-UAT browser session. Section 16's `.sfd-port-label-input` rule (added by Plan 11-03) set `background: #fff !important;` but did not set a text color, so the input inherited `#sfd-inspector { color: #eee }` — white text on white background → text was invisible on the dark navy inspector.

Fix: align `.sfd-port-label-input`'s color stack with the canonical inspector-input pattern in Section 4 (`.sfd-field input[type="text"]`) — dark navy background, light text, muted placeholder. The values are copied verbatim from the existing Section 4 rule to keep visual consistency across the inspector (no fresh color hex invented for v2.3 polish).

Sibling plan 11-07 (JS + backend) ships in the same wave and addresses GAP-11.1 / GAP-11.2 / GAP-11.3 / GAP-11.5. The two plans touch zero overlapping files and have no dependency between them.

Purpose: Make engineer-authored port labels readable as soon as they're typed. PORT-03 specifies the dropdown-or-custom label flow; UAT showed the custom-typed path is functionally working but visually broken. This plan unblocks the UAT verification of PORT-03 by making the text actually legible.

Output:
- ~6 net new lines in `planner/static/planner/css/signal_flow.css` (3 amended properties on `.sfd-port-label-input`, 1 new `::placeholder` rule).
- Zero JS edits. Zero backend edits. Zero new dependencies.
</objective>
