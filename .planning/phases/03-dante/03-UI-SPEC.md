> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

## Design Principles

Inherited from Phase 1 + Phase 2, with one Phase 3 addition:

1. **Status at a glance.** Status indicators must be readable at 2 meters from a laptop screen without zooming.
2. **No false urgency.** Visual hierarchy must prevent amber from reading as critical and red from blending into the background.
3. **Minimal cognitive load.** An engineer in show mode must not hunt for information.
4. **Mode clarity.** When the engineer has suppressed alerts (Setup/Wrap mode), the UI must make that suppression impossible to miss.
5. **Advisory honesty.** Clock status data has LOW protocol confidence. The UI must present clock data as informational without implying hardware-verified accuracy. The advisory footnote is always visible — it is not dismissible.

## Status Indicator Contract

Phase 1 and Phase 2 status dot contracts remain unchanged and apply to Dante device-level status dots.

Phase 3 adds clock role badges (NOT dots — inline text badges):

| Clock Role | Badge Style | Text Color | Background |
|------------|-------------|------------|------------|
| Master | Solid badge | `--bg-base` (#0f0f1a) | `--accent-green` (#00e676) |
| Locked | Outline badge | `--accent-green` | `rgba(0, 230, 118, 0.12)` |
| Unlocked | Outline badge | `--accent-amber` | `rgba(255, 171, 0, 0.12)` |
| Unknown | Outline badge | `--text-dim` | `transparent` |

Badge anatomy: 11px/700 uppercase text, 4px vertical padding, 8px horizontal padding, border-radius 3px. "Master" uses filled green background with dark text for maximum prominence. "Locked" and "Unlocked" use tinted backgrounds matching the Phase 2 show mode toggle tint pattern.

Ghost card status dot: uses existing `nhm-dot--unreachable` class (`--text-dim`, `opacity: 0.5`). No pulse animation.

## Component Inventory

### New in Phase 3

#### 1. Dante Card (Collapsed)

Extends existing `nhm-card` pattern. Replaces the Phase 1 latency readout with clock role badge and channel count.

Collapsed card header row:
```
[status dot] [device name]  [clock role badge]  [channel count]  [expand arrow]
```

- Status dot: 12px, standard `nhm-dot--{state}` class (from ICMP reachability, not clock status).
- Device name: `nhm-card-name`, 14px/400, `--text-primary`. Shows mDNS device name.
- Clock role badge: `nhm-clock-badge`, inline after device name, flex-shrink: 0. See badge styles in Status Indicator Contract above.
- Channel count: `nhm-card-channels`, mono 12px, `--text-secondary`, right-aligned before expand arrow. Format: `32x24` (tx x rx). If unavailable: omitted entirely (not "---").
- Expand arrow: existing `nhm-card-expand-arrow` pattern, Unicode triangle, 12px, `--text-dim`.

CSS class additions: `nhm-clock-badge`, `nhm-clock-badge--master`, `nhm-clock-badge--locked`, `nhm-clock-badge--unlocked`, `nhm-clock-badge--unknown`, `nhm-card-channels`.

Layout: flex row with `align-items: center`, `gap: 8px`. Clock badge and channel count sit between name and expand arrow. Name has `flex: 1` with `min-width: 0` and `overflow: hidden; text-overflow: ellipsis; white-space: nowrap` to prevent long mDNS names from breaking layout.

#### 2. Dante Card (Expanded)

Same `max-height` transition pattern as Phase 1/2 (`max-height: 0` to `600px`, `overflow: hidden`, `transition: max-height 0.25s ease`).

Expanded detail rows (each `nhm-card-detail-row`):
| Label | Value | Style |
|-------|-------|-------|
| IP | `169.254.x.x` | Mono 12px |
| Clock | `Master` / `Locked` / `Unlocked` / `Unknown` | Body 14px, colored to match badge |
| Channels | `TX: 32 / RX: 24` | Mono 12px |
| Last Seen | `14:32:05` | Mono 12px, `--text-dim` |

Below detail rows: "Remove from monitor" button — same `nhm-card-remove` pattern as Phase 1.

#### 3. Ghost Card (Missing Project Device)

A Dante card for a project device that was expected but not discovered on mDNS.

Visual treatment — adds `nhm-card--ghost` class alongside `nhm-card`:
- `opacity: 0.45` on the entire card.
- Status dot: `nhm-dot--unreachable` (dim grey, half-opacity — stacks with card opacity for ~0.23 effective opacity on the dot).
- Device name: shows the project device label (not mDNS name, since there is none).
- Clock role badge: `nhm-clock-badge--unknown` ("Unknown" in dim outline).
- Channel count: omitted.
- Expand arrow: hidden (`display: none`). Ghost cards do not expand — there is no data to show.
- `pointer-events: none` — ghost cards are not clickable.
- `cursor: default`.

CSS:
```css
.nhm-card--ghost {
    opacity: 0.45;
    pointer-events: none;
    cursor: default;
}
.nhm-card--ghost .nhm-card-expand-arrow {
    display: none;
}
```

Ghost cards sort AFTER live cards within the Dante grid. They appear at the end of the card grid, not interspersed with live devices.

#### 4. Advisory Footnote

Position: immediately below the Dante domain header, above the card grid. Inside `nhm-domain-body-dante`, before `nhm-grid-dante`.

Anatomy:
```
Clock status is advisory — verify on hardware before showtime.
```

Layout: full width within the domain body. Single line of text.
Font: 14px/400, `--text-secondary`.
Padding: 12px vertical, 16px horizontal.
Border-top: 1px solid `--border` (separates from domain header).
Border-bottom: 1px solid `--border` (separates from card grid).
Background: `transparent` (inherits domain section `--bg-card`).

CSS class: `nhm-dante-advisory`.

The footnote is NOT dismissible. It is always visible when the Dante section is expanded. It hides when the domain section is collapsed (it is inside the collapsible body).

#### 5. Health Check Panel

Position: inside `nhm-domain-body-dante`, below the advisory footnote, above the card grid. It is part of the Dante domain section, not a standalone section.

Layout: collapsible panel with its own header row.

**Panel header row:**
```
[checkmark or warning icon]  Pre-Show Health Check  [Re-check button]  [expand/collapse arrow]
```

- Icon: Unicode checkmark (checkmark) or warning triangle (triangle). Green checkmark when all devices present, red warning when issues found.
  - All clear: `\u2714` (heavy check mark), `--accent-green`.
  - Issues found: `\u26A0` (warning sign), `--accent-red`.
- Title: "Pre-Show Health Check", label 11px/700 uppercase, `--text-dim`.
- Re-check button: `nhm-btn nhm-btn--ghost`, 28px height, label "Re-check", positioned right-aligned before the expand arrow. `font-size: 11px; font-weight: 700; text-transform: uppercase; letter-spacing: 0.10em`.
- Expand arrow: same pattern as domain section toggle.

Panel header: `nhm-healthcheck-header`. Clickable to toggle panel expansion. `cursor: pointer`.
Panel body: `nhm-healthcheck-body`. `max-height` transition (0 to 400px, 0.25s ease).

**Auto-expand behavior:**
- On page load and after each re-check: if issues are found, panel auto-expands. If all clear, panel stays collapsed.
- Manual toggle overrides auto-expand until the next re-check.

**Panel body content (when expanded):**

Two possible states:

**State A — All Clear:**
```
All 12 Dante devices present.
```
Single line, 14px/400, `--accent-green`. No device list shown.

**State B — Issues Found:**

Result rows, one per issue:

Missing device row:
```
[red dot]  RIO3224-D2  —  Missing (expected but not found on network)
```

Unexpected device row:
```
[amber dot]  DVIO-A8  —  Unexpected (found on network but not in project)
```

Row layout: flex row, `align-items: center`, `gap: 8px`.
- Dot: 8px diameter (smaller than device status dots), `nhm-hc-dot`.
- Device name: mono 12px, `--text-primary`.
- Description: body 14px/400, `--text-secondary`.

Row styling: 36px min-height, border-bottom 1px solid `--border`. Alternating backgrounds: odd `--bg-raised`, even `--bg-card`.

CSS class patterns: `nhm-healthcheck`, `nhm-healthcheck-header`, `nhm-healthcheck-body`, `nhm-healthcheck-summary`, `nhm-hc-row`, `nhm-hc-dot`, `nhm-hc-dot--missing` (red), `nhm-hc-dot--unexpected` (amber), `nhm-hc-name`, `nhm-hc-desc`.

#### 6. Dante Rollup Pill (Update)

The existing placeholder pill `nhm-pill--placeholder` at `#nhm-pill-dante` becomes a live pill. Remove the `nhm-pill--placeholder` class when Dante data is available.

Pill content: `Dante: {online}/{total}` — same format as LA Network pill.
Color logic: same as other domain pills (all online = green, partial = amber, none = red, no data = grey/dim).

## Interaction Contract

### Dante Card Expand/Collapse

- Same behavior as Phase 1 device cards. Click anywhere on card header row to toggle.
- `nhm-card--expanded` class toggles. `max-height` transition 0 to 600px.
- `localStorage` key: `nhm_card_{device_id}_expanded` (same pattern as Phase 1).
- `aria-expanded` toggles on the card header.
- Ghost cards do NOT expand (pointer-events: none).

### Clock Status Updates

- Clock role data arrives via the same AJAX poll cycle as device status (every 3 seconds).
- Clock badge class is swapped in-place via JS: remove all `nhm-clock-badge--*` modifier classes, add the new one.
- If clock role changes (e.g., master failover), the badge updates immediately on the next poll. No transition animation — instant swap.
- Multiple devices may show "MASTER" simultaneously. The UI does not warn or interpret this (per D-06).

### Health Check Flow

1. **Auto-run on page load:** When the dashboard loads and the Dante section has data, the health check runs automatically via `GET /audiopatch/network-monitor/api/health-check/`.
2. **Re-check button:** Clicking "Re-check" triggers the same GET request. Button shows "Checking..." (disabled) during the request.
3. **Result rendering:** Response contains `{ "status": "ok"|"issues", "missing": [...], "unexpected": [...], "total_expected": N, "total_found": N }`.
4. **Auto-expand:** If `status === "issues"`, panel body expands automatically. If `status === "ok"`, panel body collapses (unless manually expanded by the user).
5. **Panel icon update:** Checkmark/warning icon in header updates based on `status`.

### Ghost Card Lifecycle

- Ghost cards appear when the health check identifies missing project devices.
- They are rendered into `nhm-grid-dante` at the END of the card grid (after all live cards).
- If a previously missing device appears on a subsequent poll, the ghost card is removed and a live card replaces it (at its sorted position among live cards).
- Ghost cards are NOT included in the rollup pill count numerator (online count). They ARE included in the denominator (total count): `Dante: 10/12` means 10 online, 12 expected.

### Dante Section State Transitions

| State | Domain Header Badge | Advisory | Card Grid | Health Check Panel |
|-------|-------------------|----------|-----------|-------------------|
| No Dante data (agent not running) | grey "0/0" | hidden | Empty state message | hidden |
| Dante data, all matched | green "{N}/{N}" | visible | Live cards only | collapsed, green check |
| Dante data, some missing | amber "{online}/{total}" | visible | Live cards + ghost cards | expanded, red warning |
| Dante data, unexpected found | amber "{online}/{total}" | visible | Live cards (unexpected in Unassigned section) | expanded, amber warning |

Note: Unexpected Dante devices are shown in the **Unassigned** domain section (per D-02), not in the Dante section. They appear in the health check panel results only.

## Accessibility Minimums

Inherited from Phase 1 + Phase 2, with Phase 3 additions:

- Clock role badges: `aria-label="Clock role: {Master|Locked|Unlocked|Unknown}"` on each badge element.
- Channel count: `aria-label="Channels: {tx} transmit, {rx} receive"` when present.
- Ghost cards: `aria-label="{device name}: missing from network"`. Despite `pointer-events: none`, screen readers can still read the card content.
- Advisory footnote: `role="note"` so screen readers identify it as supplementary information.
- Health check panel header: `aria-expanded="true|false"` toggles on click.
- Health check result dots: `aria-label="missing"` or `aria-label="unexpected"` on each 8px dot.
- Re-check button: `aria-label="Re-check Dante health"`.
- All new buttons: visible focus ring via `outline: 2px solid var(--accent-blue); outline-offset: 2px`.
