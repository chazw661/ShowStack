# Phase 1 — UI Design Contract

> Visual and interaction contract for the Network Health Monitor dashboard (Foundation phase).
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — plain HTML/CSS/JS, no frontend framework |
| Icon library | Unicode symbols inline (▲ ▼ ●) — no icon font dependency |
| Font | -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif |

---

## Design Principles

This is a **live show monitoring dashboard** viewed in dark control rooms. Three rules govern every visual decision:

1. **Status at a glance.** Status indicators must be readable at 2 meters from a laptop screen without zooming.
2. **No false urgency.** Visual hierarchy must prevent amber from reading as critical and red from blending into the background.
3. **Minimal cognitive load.** An engineer in show mode must not hunt for information. The domain rollup bar answers "is the show OK?" before the engineer scrolls.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Status dot gap from label, inline icon padding |
| sm | 8px | Gap between domain pills in rollup bar, button gap |
| md | 16px | Card inner padding, header padding sides |
| lg | 24px | Section gap between domain groups, header padding top/bottom |
| xl | 32px | Page horizontal margin on wide viewports |
| 2xl | 48px | Gap below rollup bar before first domain section |
| 3xl | 64px | Not used in Phase 1 |

Exceptions:
- Status dots: 12px diameter (not a spacing value — a size value, derived from legibility requirement)
- Touch targets for collapse toggles: minimum 44px height (accessibility)
- Alert banner: full viewport width, 0 horizontal margin, 12px vertical padding

|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Device name, IP address, timestamp text |
| Label | 11px | 600 | 1.0 | Section headings, meta labels (uppercase + letter-spacing: 0.10em) |

## Color

All values match the established mic_tracker.html CSS custom properties. The network monitor uses the same token set with no new hues introduced.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--bg-base` | `#0f0f1a` | Page background, full-page root |
| Secondary (30%) | `--bg-card` | `#181828` | Device cards, domain section backgrounds, rollup bar |
| Raised surface | `--bg-raised` | `#1e1e32` | Expanded card detail area, alert banner bg |
| Border dim | `--border` | `#2a2a45` | Card borders, dividers |
| Border bright | `--border-bright` | `#3a3a60` | Hover borders, active card outline |
| Accent blue (10%) | `--accent-blue` | `#4a9eff` | Page title, primary CTA buttons, focus rings |
| Status green | `--accent-green` | `#00e676` | Online status dot, healthy rollup pill |
| Status amber | `--accent-amber` | `#ffab00` | Degraded/warning status dot, N=1 or N=2 flap |
| Status red | `--accent-red` | `#ff5252` | Offline status dot, critical alert, unhealthy rollup pill |

**Accent reserved for:** Page title text, primary action buttons ("Start Scan", "Confirm"), focus outlines on interactive elements. Not used on status indicators (status indicators use semantic colors only).

**60/30/10 mapping:**
- 60% — `#0f0f1a` page background
- 30% — `#181828` cards and domain sections
- 10% — `#4a9eff` (blue accent) + semantic status colors

## Status Indicator Contract

Status indicators are the primary communication mechanism. They must be unambiguous.

| State | Dot Color | Dot Pulse | Rollup Pill | Meaning |
|-------|-----------|-----------|-------------|---------|
| Online | `#00e676` | none | Green fill | Device responded on last poll |
| Flapping (N=1 or N=2) | `#ffab00` | none | Amber fill | 1–2 consecutive failures — confirming, no alert yet |
| Offline (N≥3) | `#ff5252` | slow pulse 2s | Red fill | Alert fired — confirmed offline |
| Unknown / no data | `#505070` | none | Grey fill | Device never polled or monitor not running |

Dot diameter: 12px. Dot shape: circle (`border-radius: 50%`).

Pulse implementation: CSS `@keyframes` opacity 1→0.4→1 at 2s infinite. Applied only to the Offline dot. No JS required.

## Component Inventory

### 1. Domain Rollup Bar

Position: fixed to top of `nhm-root` content area, below the page header.
Layout: horizontal flex row, single line, wraps on narrow viewports.
Content: one pill per domain — "LA Network: 8/8", "Switches: 3/3". Healthy=green fill, partial=amber, none/unknown=red.
Pill anatomy: `[dot] [Domain Name]: [online]/[total]`
Font: Label — 11px/600 uppercase.
Height: 40px.
Border: 1px solid `--border` below the bar.

Phase 1 shows: **LA Network** domain only (ICMP-only phase). Dante and Switches pills render as grey/unknown placeholders — they exist visually so the layout does not shift in Phase 2.

### 2. Domain Section

Layout: full-width block with `--bg-card` background, 1px `--border` border, border-radius 6px.
Header row: domain name (heading 16px/700) + device count badge + collapse toggle (▲/▼).
Collapsed state: header row only, device grid hidden.
Expanded state: header + device card grid.
Card grid: CSS grid, `repeat(auto-fill, minmax(260px, 1fr))`, 8px gap.
Collapse state persists in `localStorage` (key: `nhm_domain_{slug}_collapsed`).

### 3. Device Card (collapsed)

Size: min 260px wide, height auto.
Background: `--bg-raised`.
Border: 1px solid `--border`. On hover: `--border-bright`.
Content (single row): `[status dot] [device name] [latency or "—"]`
Transition: `border-color 0.15s ease`.
Click anywhere on card: expand to detail view (inline expansion, not modal).

### 4. Device Card (expanded)

Expands below the collapsed row within the same card.
Additional rows (each 14px/400):
- IP address (mono 13px)
- Last seen timestamp (mono 13px, dim color)
- Poll count / failure count
- "Remove from monitor" button (ghost, danger-colored text)

Expanded state indicated by: card border changes to `--border-bright`, background shifts to `--bg-raised`.

### 5. Alert Banner

Position: top of `nhm-root`, above domain rollup bar. Hidden when no active alerts.
Height: auto, min 48px.
Background: `--bg-raised` with left border 4px solid `--accent-red`.
Content: `[red dot pulse] [Device Name] offline — [timestamp] [Dismiss button]`
Multiple alerts: stack vertically, each as a separate banner row.
Dismiss: removes from banner. Does not suppress future alerts for that device.
Font: heading 16px/700 for device name, body 14px/400 for detail.

### 6. Session History Timeline

Layout: ordered list below the domain sections, collapsed by default.
Toggle: "Session History ▼" — 11px/600 uppercase label, click to expand.
Each event row: `[timestamp mono] [dot color] [device name] [event type] [details]`
Event types and labels:
- `ONLINE` → "came online"
- `OFFLINE` → "went offline"
- `SCAN_STARTED` → "Network scan started"
- `MONITOR_STARTED` → "Monitor started"
Row height: 36px. Alternating row: `--bg-card` / `--bg-base`.
Timestamp: mono 13px, `--text-dim`.
Max visible without scroll: 20 rows. Scroll within the timeline container.

### 7. "Not on Show Network" State

Shown when: monitor is running but ping sweep returns 0 responding devices, or NIC selection yields no active interface.
Layout: replaces domain sections entirely. Full-width message card.
Content:
- Heading: "Not connected to show network" (heading 16px/700, `--text-primary`)
- Body: "No devices responded on [NIC name / subnet]. Connect your laptop to the show network and run a new scan." (body 14px/400, `--text-secondary`)
- CTA: "Select Network Interface" button (primary blue)
Background: `--bg-card`. Border: 1px solid `--border-bright`. Border-radius: 6px. Padding: 32px.

### 8. NIC Selection / Scan Controls

Position: header action area, right side, inline with page title.
NIC selector: `<select>` styled to match `--bg-input`, `--text-primary`, `--border-bright`. Width: auto, min 160px.
"Start Scan" button: primary blue (`--accent-blue`), 11px/600 uppercase, padding 7px 14px, border-radius 4px.
"Scanning..." state: button disabled, text "Scanning…", spinner (CSS border animation, no image).
Scan result count: inline text "14 devices found — select to monitor" (body 14px, `--text-secondary`).

## Interaction Contract

### SSE Connection States

The frontend `EventSource` connection has three states that must be visually communicated:

| State | Indicator | Position |
|-------|-----------|----------|
| Connected | None (silent) | — |
| Reconnecting | "Reconnecting…" amber text, 11px dim | Bottom-right corner, fixed |
| Failed (5 retries) | "Live updates paused — reload page" amber banner | Below rollup bar |

No spinner on connected state — silence = healthy for the SSE connection.

### Expand/Collapse Behavior

- Device card expand: CSS `max-height` transition from 0 to auto (use `max-height: 0` → `max-height: 500px`, `overflow: hidden`, `transition: max-height 0.2s ease`).
- Domain section collapse: same `max-height` pattern on the card grid container.
- State stored in `localStorage`. Key pattern: `nhm_domain_{slug}_collapsed` and `nhm_card_{device_id}_expanded`.

### Alert Dismiss Behavior

Clicking "Dismiss" on an alert banner: removes the banner row with a `opacity 0` + `max-height 0` CSS transition (0.2s). Does NOT reset the failure counter or change the device card status dot. The device card remains red until the device comes back online.

### Discovery Flow (Scan UI Interaction)

1. Engineer selects NIC from dropdown.
2. Clicks "Start Scan".
3. Button becomes disabled "Scanning…" with spinner.
4. Results appear as a selectable list below the scan controls: checkboxes + device rows (IP, hostname if resolved, latency).
5. "Add to Monitor" button activates when at least one device is checked.
6. Clicking "Add to Monitor" POSTs selections and refreshes the domain section.

No modal. Scan results appear inline, below the header controls, above the domain sections.

## Accessibility Minimums

- Color is never the sole status indicator. Each status dot is accompanied by a text label in expanded view (online / offline / confirming).
- `aria-label` on every status dot: `aria-label="{device_name}: {status}"`.
- Alert banners use `role="alert"` so screen readers announce them.
- All buttons have visible focus rings: `outline: 2px solid var(--accent-blue); outline-offset: 2px`.
- Contrast ratio for body text on background: `#e8e8f0` on `#0f0f1a` = 14.9:1 (WCAG AAA).
