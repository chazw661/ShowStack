## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Django template -> rendered HTML | Engineer-controlled fields (session.name, session.notes, track.label_override, track.notes) cross into the page DOM |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | XSS (label / name / notes) | All template renders of user-controlled fields | mitigate | Django template auto-escape applies to `{{ ... }}` by default. The plan explicitly forbids `\|safe` on session.name, session.notes, track.label_override, track.notes. The single legitimate `\|safe` use is `picker_data_json|safe` — already-`json.dumps()`'d server-side (Plan 03), so it's a JSON literal, not user-controlled string. |
| T-05-02 | XSS via onclick attribute | session.name passed to `mtsRenameSession` etc. | mitigate | All inlined string args use `|escapejs` filter (Django's escape-for-JS-context). A name like `'); alert(1); //` becomes `'); alert(1); //` — safe inside a single-quoted JS string. |
| T-05-03 | XSS via Content-Disposition (file download from editor button) | href="{% url 'planner:multitrack_export_rpp' session.id %}" | mitigate | Plan 04's `_safe_filename` slugifies the session name before it hits Content-Disposition. The template doesn't construct this header. |
| T-05-04 | XSS via inline color style | track.color_override rendered into a `style="background-color: ..."` attribute | mitigate | The plan explicitly does NOT inline color into a `style` attribute. JS sets the visual color via `setProperty('background-color', ..., 'important')` AFTER reading `data-color="{{ track.color_override }}"`. Even if a malicious value somehow bypasses Plan 04's hex validation and reaches the template, it lands in a `data-` attribute (where it's auto-escaped) — not in a CSS context. |
| T-05-05 | Clickjacking on destructive actions | Delete-session button on dashboard cards | accept | The custom dashboard renders inside Django admin's authenticated shell. Django sets `X-Frame-Options: DENY` (or SAMEORIGIN) by default. Standard Django middleware mitigation — no template-level work needed. |
| T-05-06 | CSRF on AJAX-via-onclick | Inline onclick="mtsRemoveTrack(N)" calls | mitigate | The JS function reads CSRF from the hidden `<form style="display:none">{% csrf_token %}</form>` and includes the token in the fetch() X-CSRFToken header. Django middleware enforces. |
| T-05-07 | Information disclosure (cross-project link tampering) | URLs constructed via {% url 'planner:multitrack_editor' session.id %} | accept | Server-rendered URLs use the `session` object from the project-scoped queryset (Plan 03's `multitrack_dashboard` filters by current_project).
