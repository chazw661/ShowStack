# Phase 1 — UI Design Contract

> Visual and interaction contract for the Multitrack Session Builder module. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

This phase ships the entire user-visible surface of the module: dashboard (session list), track editor, channel-picker modal, capacity bar, and the "Export to Reaper" affordance. The contract below is binding for the planner and executor.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Django admin theme is the canonical UI shell |
| Preset | not applicable |
| Component library | none — hand-rolled DOM, scoped CSS class prefix `mts-` |
| Icon library | Unicode glyphs + emoji (matches existing modules: `📊`, `🔧`, `▶`, `└`, `🔍`, `⊕`, `×`) — no SVG icon framework |
| Font | inherit from Django admin (`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, …`); JetBrains-style monospace `font-family: monospace` only for channel numbers and badges |
| Theme | Dark, locked in `admin/base_site.html` via `data-theme="dark"`; light mode is explicitly hidden by `#django-admin-theme-toggle { display: none }` |
| JS stack | jQuery (already loaded by Django admin) + Sortable.js for TRK-05 drag-reorder. No HTMX, no Alpine, no React. |
| CSS override pattern | All visual rules use `!important` because Django admin uses `!important` pervasively. JS CSS writes use `el.style.setProperty('color', value, 'important')` — direct `el.style.color = value` will silently fail. |
| Module CSS class prefix | `mts-` (mirrors `cc-` for Comm Config, `mt-` for Mic Tracker) |
| Module URL | `/audiopatch/multitrack/` |

**Closest UX precedent:** `templates/planner/comm_config.html` — same dark shell, tab bar, card grid, modal overlay pattern. Implementer should diff this UI-SPEC against `comm_config.html` to confirm visual continuity.

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap, badge padding-y, drag-handle hit margin |
| sm | 8px | Compact element gap (track row internals, picker checkbox row) |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, header bottom margin |
| xl | 32px | Layout gaps between major regions (header → editor body) |
| 2xl | 48px | Empty-state vertical padding |
| 3xl | 64px | Page-level spacing (rare; reserved for empty dashboard state) |

**Component-specific applied spacing (binding):**

| Element | Padding | Gap |
|---------|---------|-----|
| Page container (`.mts-container`) | `lg` (24px) outer, `max-width: 1400px` | — |
| Header bar (`.mts-header`) | bottom margin `lg` (24px) | item gap `sm` (8px) |
| Session card (`.mts-card`) | `md` (16px) all sides | grid gap `md` (16px) |
| Track row (`.mts-track-row`) | `sm` (8px) vertical, `md` (16px) horizontal | inline gap `sm` (8px) |
| Picker modal body | `md` (16px) | — |
| Picker channel row (`.mts-pick-row`) | `xs` (4px) vertical, `sm` (8px) horizontal | inline gap `sm` (8px) |
| Modal overlay container | `md` (16px) viewport inset | — |
| Tab bar (`.mts-tabs`) | each tab `8px 16px` | — |
| Buttons (`.mts-btn`) | `8px 16px` | — |
| Capacity bar (`.mts-capacity`) | `sm` (8px) vertical, `md` (16px) horizontal | — |

**Exceptions (carried from existing module precedent — do not re-declare globally, but they are accepted for parity):** existing modules sometimes use `6px`, `10px`, `14px` inside compact card internals (`comm_config.html` line 102, 122). The Multitrack module will NOT introduce new odd values — stick to the 8-point scale above. If matching an inline cell from `comm_config` by sight is required (e.g. "card density should match"), use `sm` (8px) and accept a 2px visual difference.

## Typography

The Multitrack module inherits the Django admin font stack and declares **exactly 4 sizes and 2 weights** for the new component classes.

| Role | Size | Weight | Line Height | Used For |
|------|------|--------|-------------|----------|
| Body | 14px (0.875rem) | 400 (regular) | 1.5 | Track-row label, picker rows, dropdown items, secondary text |
| Label | 12px (0.75rem) | 600 (semibold) | 1.4 | Tab labels, button labels, capacity readout, badge text, table column headers, section captions (uppercase tracking 0.06em) |
| Heading | 20px (1.25rem) | 600 (semibold) | 1.2 | Editor session title, modal title, dashboard section titles |
| Display | 28px (1.75rem) | 600 (semibold) | 1.2 | Empty-state heading on the dashboard ("No sessions yet") and major page titles |

**Monospace exceptions (accepted, not new sizes):** channel numbers (`IN 1`, `IN 47`), source-type badges, hex color codes, and capacity ratios (`47 / 64`) render in `font-family: monospace`. Monospace exceptions render at Label (12px) or Body (14px) with weight 600. This matches `cc-port-gid`, `cc-port-badge`, `cc-key-letter` in `comm_config.html`.

**Weight rule:** No italics. No oblique. Two weights only: 400 and 600.

**Line-height rule:** All multi-line copy (notes textarea, error messages, empty-state body) uses 1.5. All short single-line labels and headings use 1.2–1.4.

## Color

The module uses the established ShowStack dark palette. Roles below are **binding**; do not introduce new accents.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0a0a` | Page background, modal overlay backdrop fill (with 0.6 alpha) |
| Surface 1 (20%) | `#12122a` | Cards (session card, track row, picker channel row), modal body |
| Surface 2 (10%) | `#1e1e3a` | Raised inputs (selects, hex pickers), tab content panel, badge background |
| Border default | `#2a2a4a` | Card border, tab divider, input border |
| Border focus | `#4a9eff` | Input focus ring, active tab underline, hovered card border |
| Body text | `#e0e0e0` | Resolved track labels, primary copy, headings |
| Secondary text | `#c0c0c0` | Button copy, dropdown items, secondary metadata |
| Caption text | `#8888aa` | Captions, helper text, "no source" placeholder |
| Label text (uppercase) | `#a0a0cc` | Section labels, picker tab counts, table column headers |
| Accent (10%) | `#4a9eff` (ShowStack blue) | **Primary CTA fill, active tab underline, focus ring, drag-handle hover, "Add N selected" button, current-session highlight** |
| Save / Success | `#00ff88` | "Save" / "Export to Reaper" button fill (mirrors Django admin `.default` button) — text color on this fill is `#000` for contrast |
| Destructive | `#dc3545` | Delete-session button, "Remove track" `[×]` hover state, capacity-over warning fill |
| Warning amber | `#f0a500` | Reserved — phase 1 does not use; available for future "approaching capacity" tween (do not implement in phase 1) |

**Accent (`#4a9eff`) is reserved for, and ONLY for:**
1. The "Add N selected" primary commit button inside the picker modal
2. The "+ New Session" primary CTA on the dashboard
3. The active-tab underline in the picker tab bar (`[Inputs] [Aux] [Matrix] [Stereo] [Manual]`)
4. The active-tab underline in the editor (if a tabbed editor is used) — otherwise N/A
5. Input focus ring (`outline: 2px solid #4a9eff` + `box-shadow: 0 0 10px rgba(74,158,255,0.3)`)
6. Drag-handle hover state and the hover border on session cards
7. Current-session breadcrumb link

**Save (`#00ff88`) is reserved for, and ONLY for:**
1. The "Export to Reaper" download button on the editor toolbar
2. The "Save" button on the session-metadata edit form (if rendered as a Django admin change form, this is automatic)
3. Save-confirmation success messages

**Destructive (`#dc3545`) is reserved for, and ONLY for:**
1. The dashboard "Delete session" action in the per-card dropdown (matches `comm_config.html` `cc-btn-danger`)
2. The capacity-bar fill when `current_count > recorder_capacity`
3. The `[×]` "Remove track" button hover state on a track row

**Color swatch rendering rule (TRK-04):** A track's `color_override` is rendered as a 16×16 rounded-corner swatch (`border-radius: 4px`) with a 1px `#2a2a4a` border. When `color_override` is empty, render an outline-only swatch (no fill, dashed `#3a3a5a` border, 16×16) — DO NOT render any placeholder color. The swatch is the click target for the color picker.

**Track-source-type badge color coding (binding):**

| `source_type` | Badge background | Badge text | Badge border |
|---------------|------------------|------------|--------------|
| `input` | `#1a3a4a` | `#4a9eff` | `#2a4a6a` |
| `aux` | `#1a3a1a` | `#4acc4a` | `#2a5a2a` |
| `matrix` | `#3a2a1a` | `#f0a500` | `#5a4a2a` |
| `stereo` | `#2a1a3a` | `#a070ff` | `#4a3a6a` |
| `manual` | `#2a2a2a` | `#c0c0c0` | `#444444` |

These mirror the cc-port-badge convention in `comm_config.html` (lines 469–470). Badge content is the uppercase short token: `INPUT`, `AUX`, `MTX`, `ST`, `MANUAL`. Font: monospace 12px weight 600.

```
   │  │  [⊘]                    [                   ] │   │
   │  └────────────────────────────────────────────────┘   │
   │                                                       │
   │  [+ Add another]                  2 manual queued    │  ← + Add another (#2a2a4a secondary), queued count caption 12px
   ├───────────────────────────────────────────────────────┤
   │              [Add 2 selected]    [Cancel]             │  ← combined commit; if other tabs also have selections, count combines
   └───────────────────────────────────────────────────────┘
```

**Tab counts (D-09):** Each non-Manual tab label includes a parenthetical count of channels NOT already in the session. Counts update live when the user opens the picker, but DO NOT update reactively if the underlying console changes mid-session — the picker reads a snapshot on open.

**Tab-active rule:** First tab with available channels is active by default. On a fresh new session (D-12), the picker opens auto-active on Inputs.

**Filter behavior:** Applies only to the active tab's list. Case-insensitive substring match against the channel's display number AND its source/name field. Filter does NOT cross tabs.

**Already-added rule (D-09):** Channels already represented by a `MultitrackTrack` (where `source_type` and `source_id` match) are filtered OUT of every non-Manual tab's list at picker-open time. They are not greyed out — they are absent. The Manual tab does NOT filter (manual tracks have no source ID to match against).

**Selection-state persistence:** Switching tabs preserves selections within a session (Inputs tab keeps its checks even after the user clicks Aux). Tab switch does NOT lose state. Cancel button discards all selections without commit; X button at top-right also cancels.

**Commit count rendering:** The "Add N selected" button text recomputes on every checkbox toggle and on Manual-tab queue change. N is the SUM of (checked rows in non-Manual tabs) + (queued manual rows with non-empty Label). Validation: if any queued manual row has empty Label, button shows `Add N selected` enabled but the commit handler returns the user to the Manual tab with the empty-Label rows highlighted in red border and the inline error copy beneath each.

**Append rule (D-10):** New tracks are appended in the order: all Inputs (in checkbox order top-to-bottom), then all Aux, then Matrix, then Stereo, then Manual queue (in queue order). `track_number` values are assigned at commit by extending from `MAX(track_number) + 1` on the existing session, sequentially.

### Drag-Reorder Behavior (TRK-05)

- Sortable.js initialised on `.mts-track-list` with `handle: '.mts-drag'`, `animation: 150`.
- On `onEnd`, send a `POST /audiopatch/multitrack/<session_id>/reorder/` with the new ordered list of track IDs.
- Optimistic UI: numbers (`#1`, `#2`, …) recompute client-side immediately on drop. If the XHR fails, revert order and show the network-error toast.
- Renumbering: on every reorder save, server assigns dense `track_number` values 1..N. Sparse numbering is NOT supported.

### Bulk Type Toggle Removal (per D-08)

The editor does NOT show top-level "Include all Aux / Matrix / Stereo" toggles. TRK-09 is satisfied entirely inside the picker via per-tab Select-all/Clear. This is a deliberate departure from the spec's "collapsible section toggles" wording. The picker is the single source of truth for set composition.

## Accessibility Notes (binding minimum)

- All form inputs have `<label>` elements (visible or `sr-only`).
- All icon-only buttons (drag handle, remove, notes pencil) have `aria-label`.
- Color swatch states are conveyed by border style as well as color (dashed for empty, solid for set) so colorblind users can still tell.
- Capacity-over warning is conveyed by both color (red fill) AND text suffix (`— 8 over capacity`), not color alone.
- Modal traps focus while open (basic `tab` cycling within the modal panel).
- `Escape` closes the modal (matches the help modal pattern in `base_site.html` line 547).
- Track-row drag is keyboard-accessible: arrow up/down on a focused drag handle moves the row by one position. (If Sortable.js doesn't ship this out of the box, the executor adds it as a small jQuery handler — TRK-05 has no a11y requirement in REQUIREMENTS.md but minimum-viable keyboard support is the bar.)
