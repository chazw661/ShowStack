```python
@staff_member_required
def multitrack_export_rpp(request, session_id):
    """GET: download a Reaper .RPP file for this session (RPP-01..04).

    Returns text/plain attachment. Filename: <safe(session.name)>.RPP.
    """
    current_project = getattr(request, 'current_project', None)
    if not current_project:
        return redirect('/')

    session = MultitrackSession.objects.filter(
        id=session_id, project=current_project
    ).select_related('console').first()
    if not session:
        return redirect('planner:multitrack_dashboard')

    if not _has_enabled_tracks(session):
        enabled_tracks_qs = session.tracks.filter(enabled=True).order_by('track_number')
        return render(
            request, 'planner/multitrack/editor.html',
            _editor_context(
                session, tracks=enabled_tracks_qs,
                current_project=current_project,
                export_error='This session has no enabled tracks. '
                             'Enable at least one track to export.',
                auto_open_picker=False,
            ),
        )

    body = build_rpp(session)
    response = HttpResponse(body, content_type='text/plain; charset=utf-8')
    response['Content-Disposition'] = (
        f'attachment; filename="{_safe_filename(session.name)}.RPP"'
    )
    return response
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP request → view function | Auth gate (`@staff_member_required`) closes the unauthenticated case. Project scope (`filter(id=session_id, project=current_project)`) closes IDOR. |
| view → `build_nlpr(session)` | Pre-filtered ORM instance is passed in; exporter trusts its input per the module docstring. |
| `build_nlpr` raises → response shape | `ExportTemplateError` is the ONLY exception the view catches explicitly. Unexpected exceptions bubble to Django's default 500 handler (acceptable — same posture as Phase 1's `multitrack_export_rpp`). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-13 | Spoofing | Unauthenticated request | mitigate | `@staff_member_required` decorator (Django's stdlib admin decorator — redirects to admin login). Matches Phase 1 download views verbatim. |
| T-04-14 | Tampering | URL `session_id` belongs to another tenant (IDOR) | mitigate | Combined filter `filter(id=session_id, project=current_project)` returns None for any session not in the engineer's current project; view returns 302 to dashboard. |
| T-04-15 | Tampering | Header injection via session name in `Content-Disposition` | mitigate | `_safe_filename(name)` strips CRLF / control chars (verified at `planner/views.py:6856-6868`). Reused verbatim. |
| T-04-16 | Repudiation | No audit trail of who downloaded what | accept | Phase 1 doesn't log downloads either; out of scope for v4.0. |
| T-04-17 | Information Disclosure | Cross-tenant data leak via response body | mitigate | The session is pre-filtered by `project=current_project`; `build_nlpr` is a pure function of the filtered session. No additional tenants' data is reachable. |
| T-04-18 | Denial of Service | Repeated downloads of huge sessions | accept | RESEARCH §Security: ~30-50MB peak for a 1000-track session. Acceptable for solo engineer download endpoint on Railway. |
| T-04-19 | Elevation of Privilege | Bypassing `@staff_member_required` | mitigate | Decorator is part of Django stdlib; no path bypasses it. |
</threat_model>
