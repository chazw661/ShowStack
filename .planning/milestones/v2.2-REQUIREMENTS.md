# Milestone v2.2 Requirements — Signal Flow Diagrammer (ARCHIVED)

**Defined:** 2026-05-19
**Shipped:** 2026-05-22
**Driver:** GitHub issue #13
**Core Value:** ShowStack knows your patch, your labels, and your gear; once entered, that data drives every export your show needs.

Source: `.planning/research/SUMMARY.md` (2026-05-19) is the research-grounded basis for this scope. The locked technology stack is `@joint/core` 4.2.4 (MPL-2.0) for the canvas, `html-to-image` 1.11.11 (MIT) for PNG export, zero new Python dependencies. The single source of truth for canvas state is a `JSONField` blob on the `SignalFlowDiagram` model; smart-shape equipment references live inside the JSON via `GenericForeignKey` keys (`content_type_id`, `object_id`), enriched server-side on load.

## v2.2 Requirements

### Diagram Management — DGM

- [x] **DGM-01**: User can see all signal-flow diagrams for the current project on a list page, with create / rename / delete actions. _(Phase 7, validated 2026-05-20)_
- [x] **DGM-02**: User can create a new diagram by entering a name; diagram is scoped to the current project via `CurrentProjectMiddleware`. _(Phase 7, validated 2026-05-20)_
- [x] **DGM-03**: User can rename a diagram from the list page; name is unique per project. _(Phase 7, validated 2026-05-20)_
- [x] **DGM-04**: User can delete a diagram from the list page; deletion removes the canvas state and all node/connector references. _(Phase 7, validated 2026-05-20)_
- [x] **DGM-05**: All diagram views enforce `.filter(project=request.current_project)` on every lookup; cross-project access returns 404. _(Phase 7, validated 2026-05-20)_
- [x] **DGM-06**: Canvas changes autosave on idle within 2.5 s; the editor shows a "Saved" / "Saving…" / "Failed — retry" status indicator. _(Phase 9, validated 2026-05-22 — actual debounce floor is 1500 ms, well under spec)_
- [x] **DGM-07**: Concurrent edits from a second tab return HTTP 409 on save; the losing tab shows a non-dismissable banner: "Diagram was modified elsewhere — reload to see latest." _(Phase 9, validated 2026-05-22 — atomic `filter(version=loaded).update(version=F+1)` lock, banner reveal + canvas lockout)_
- [x] **DGM-08**: Closing the tab or navigating away triggers a `keepalive: true` final save if there are unsaved changes. _(Phase 9, validated 2026-05-22 — `visibilitychange` + `pagehide`; no `beforeunload`, no `sendBeacon`)_

## Out of Scope (deliberately excluded — final)

- **Auto-layout** (hierarchical / force-directed): destroys physical topology.
- **Real-time multi-user collaborative editing**: last-write-wins + HTTP 409 is correct for solo A1 workflow.
- **SVG export as an artifact**.
- **Pictographic rack-unit faceplates per model**.
- **Console families outside Yamaha CL/QL/Rivage PM** (project-level scope).
- **Network monitoring inside ShowStack** (project-level scope; v1.0 scrapped).

## Constraints (held throughout v2.2)

- ✅ No new Python dependencies. `models.JSONField` + `django.contrib.contenttypes` (built-in).
- ✅ `@joint/core` is MPL-2.0 (not MIT). Shipped unmodified `joint.min.js` + `THIRD_PARTY_LICENSES.txt`.
- ✅ System fonts only on shape + connector labels (prevents PNG canvas-taint on cross-origin webfonts).
- ✅ Additive migrations only. Two migrations: `0158_signalflowdiagram` (Phase 7) — no edits to existing tables.
- ✅ Project scoping enforced on every view. IDOR walks on autosave; `request.current_project` filter on all GETs.
